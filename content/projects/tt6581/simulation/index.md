## Overview

The TT6581 has two verification environments:

1. **Verilator C++ testbenches** — fast cycle-accurate simulation
2. **CocoTB Python testbenches** — high-level functional tests, also run on the gate-level netlist

## Verilator Testbenches

All Verilator testbenches live in `sim/` and are built with `make <target>`:

```bash
make tt6581           # 10-second demo song → WAV
make tt6581_player    # Full Monty on the Run → WAV
make tt6581_bode      # Sine sweep → Bode plot
make svf              # SVF frequency response (all 4 modes)
make envelope         # ADSR envelope capture
make delta_sigma      # Sine → PDM → reconstruction + FFT
make mult             # Multiplier randomised test
make spi              # SPI read/write test
```

### PDM Reconstruction Pipeline

Several testbenches capture the 1-bit PDM output as packed binary files. A Python script then reconstructs the audio:

1. **Unpack** — `np.unpackbits` converts packed bytes to a stream of 0/1 samples
2. **Map** — `0 → -1, 1 → +1` to recover the bipolar PDM signal
3. **Filter** — 4th-order Bessel low-pass via `scipy.signal.bessel` + `sosfilt`
4. **Downsample** — ÷200 to convert from 10 MHz PDM rate to 50 kHz audio rate
5. **Normalise & save** — Write to `.wav` at 50 kHz sample rate

### Key Testbenches

#### tt6581 — Demo Song

Plays a 10-second composition using all three voices and the SVF filter, demonstrating waveform selection, ADSR envelopes, frequency changes, and filter sweeps. The PDM output is captured and converted to a WAV file.

#### tt6581_player — Monty on the Run

Replays the entirety of Rob Hubbard's *Monty on the Run* by reading a stimulus file of SID register writes. The stimulus was generated by running the original 6502 assembly in a modified emulator that records all memory writes to the SID address range, then translating them to TT6581 register addresses.

- Input: `stimulus/Hubbard_Rob_Monty_on_the_Run_tt6581_stimulus.txt`
- Output: 360-second WAV file
- Simulation time: ~15 minutes on a modern machine

#### tt6581_bode — Frequency Response

Sweeps a sine wave through the audio range with the SVF configured as a 1 kHz low-pass filter. The output is captured and analysed with a Python script to produce a Bode magnitude plot, verifying the filter's frequency response against the expected -12 dB/octave rolloff.

#### delta_sigma — Modulator Analysis

Inputs a clean 1 kHz sine wave to the delta-sigma module and captures the PDM output. The Python analysis script produces:

- A time-domain plot showing 3 cycles of the reconstructed waveform
- An FFT showing the noise floor and the degree of noise shaping

## CocoTB Testbenches

Three Python-based tests run automatically as GitHub Actions on every push:

1. **Waveform test** — Verifies all four waveform types produce the expected shapes
2. **Frequency test** — Checks that the output frequency matches the programmed FCW
3. **Envelope test** — Validates ADSR timing against expected attack/decay/release curves

These tests also run against the **synthesised gate-level netlist** (post-OpenLane) to catch any synthesis or timing issues.

## Directory Structure

```
sim/
├── cpp/         # C++ testbench sources
├── tb/          # SystemVerilog wrapper modules
├── scripts/     # Python analysis & plotting scripts
├── stimulus/    # Stimulus files for the player testbench
├── logs/        # VCD waveform dumps
├── tmp/         # Intermediate binary data
├── out/         # Final outputs (WAV, PNG plots)
└── Makefile
```
